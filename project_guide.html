Project Guide

This class involves a cumulative project where you are required to:

  1. Identify a multivariate analysis problem that is of interest to you; although the problem must involve multiple variables, your analysis can analyze a single factor at a time.
  2. Gather data source(s), determine the method of data collection and provenance of the data,
  3. Clean the data, reporting on what data was removed, augmented, or modified from the data set, why and how it was treated
  4. Possibly transform the data, including adding factors that are combinations of other factors (e.g., perhaps involving computing state-wide summaries from a county-by-county table)
  5. Perform a linear regression, logistic regression, or other classification methods discussed in class.
  6. [Deliverable 1] Write a clear analysis of your problem, ideally structured as a Jupyter notebook with inter-mixed data analysis and narrative describing the problem
  7. [Deliverable 2] a short 10-15 minute video using free methods summarizing your problem, analysis methods, and results.
  8. [Deliverable 3] Create a GitHub repository with your work and put it on display for future employers to see.

Here is the project structure:

  1. PART 1 At this stage, you will work on your Project Topic and go through the initial data cleaning and an exploratory data analysis. You can choose a regression or classification study. If your dataset is not too big (e.g., less than 100,000 samples), it will probably suffice to read the data into a Pandas data frame and describe aspects of the data. In this phase, it's ok to have a few (e.g., 1 or 2) ideas and data.
    • Part 1 deliverable: Initial Jupyter Notebook: Submit a Jupyter notebook with Project Topic and Data Cleaning/EDA sections. Review the Part 1 Rubric for more detailed specifications.
  1. PART 2 In part 2, you finalize your idea or strengthen the original approach based on the feedback you received from the class.
    Now is the time to finalize your Project Topic decision.
    If you had more than one idea for Part 1, it is time to choose a single idea. If you anticipate that the idea you initially used for part 1 will be challenging to find suitable data (e.g., the right kind of data for the task, not too big, not too small, not too much missing data, etc.), you can use a different idea for Part 2 than you used for Part 1.
    Part 2 builds on the same rubric as Part 1, but we'll focus more on EDA and data cleaning this time. You will also plan your model approach.
    • Part 2 deliverable: revised Jupyter Notebook: Submit a revised version of the Jupyter notebook for Part 1 or a new notebook if you choose a new idea. Look at the Project 2 rubric for details.
  1. PART 3

  • Model training and main analysis- anticipated timeline: ~1.5 - 2 weeks. Start the main analysis (main analysis refers to machine learning such as classification or regression, prediction or inference etc OR other stat analysis). Depending on your tasks, you may have one model or more. Generally, it is considered a higher-quality project if you compare multiple models to show your understanding of why a particular model works better than others or what limitations or cautions certain models may have. For machine learning models, show enough effort on the hyperparameter optimization). We recommend using the Part 3 rubric as a guide for this section. Project Resource

Choosing data for regression or classification tasks

Due to time concerns, you will likely start shopping for data before we learn about regression and classifications in machine learning. So here are some basic concepts and simple suggestions on what to look for when you browse data.

  • Types of data, size, and shapes: You may encounter various kinds of data, such as tabulated data (meaning the data is a table format), images, structured and/or unstructured texts, sound files, etc. At some point, other forms of data BECOME tabular data; that is where you will be doing your cleaning and analysis. If you are concerned about the difficulty of working with other dataset types, the easiest option is to start with tabulated data.

  • Tasks (labels): Predictive machine learning (meaning we want to predict something given data) has two popular tasks- regression and classification. The regression task is to predict a value for a continuous variable. However, regression can also be used for a discrete variable if the discrete variable is ordinal (the order has meaning). The classification task is to predict a categorical (discrete) variable. "Binary classification" refers to classifying instances into one of two classes (e.g., 0 or 1). "Multiclass classification" refers to classifying cases into one of three or more classes. Although you can try multiclass classification, it would be more manageable to do binary class classification with the methods we learned in this course. Advanced machine learning or deep learning methods are outside the scope of this course.

  • A good example of a regression task is predicting house prices given features such as square footage, number of bedrooms, number of bathrooms, location (e.g., zip code), build material, etc. Sometimes, you need to be careful if the data is a time series where the target variable changes over time. For example, if the data contains historical data on the same house (or product), the data can be considered a time series. In an extreme case where the data only has one item, or a few items, and their price history over ten years, it is more suited to a time-series approach than the linear regression model we will be learning in this class. On the other hand, if the data has something like 10,000 different houses with features in the same year or a few years when the housing market was more or less the same, it can be addressed well with typical regression methods.

  • Another thing to consider before applying a method is whether the types of the data features are suitable for the choice of the model. For example, some models, such as (multi-) linear and logistic regression, are best when the features are continuous or ordinal discrete. Although the fitting can still work, the values of non-ordinal categories are not meaningful in the regression setting. On the other hand, if the data has a significant number of categorical values in the features, then nonparametric methods such as tree methods and similar would work better.

  • For classification, try finding data with more or less balanced labels. Suppose the data has close to 50:50 for positive and negative labels for binary class or approximately equally distributed labels among N classes. In that case, you will have an easier time training than having 10:1 or 100:1 imbalanced labels. With some caution, you can still try a dataset with a label imbalance (e.g., if you find the data very interesting). If the data is large enough, you can use techniques like undersampling the majority, oversampling the minority, or data synthesis, so imbalanced data will still work reasonably well.

EDA procedure

  • Describe the data sources, the hypothesis, or the problem you wish to analyze, and then describe the factors or components that make up the dataset (The "factors" here are called "features" in machine learning. These factors are often columns in the tabulated data). For each factor, use a box plot, scatter plot, histogram, or KDE plot to describe the distribution of the data as appropriate.
  • Describe correlations between different factors of the dataset and justify your assumption that they are correlated or not correlated. You may use numeric or qualitative/graphical analysis for this step.
  • Determine if any data needs to be transformed. For example, if you're planning on using a KNN method for prediction, you may need to normalize or scale the data if there is a considerable difference in the range of the data.
  • Using your hypothesis, indicate if it's likely that you should transform data, such as using a log transform or other transformation of the dataset. For example, suppose your hypothesis concerns classifying customers' purchasing decisions based on income levels. In that case, you may want to examine the incomes to determine if they follow a log, polynomial or other distribution.
  • You should determine if your data has outliers or needs to be cleaned. Are there missing data values for specific factors? How will you handle the data cleaning? Will you discard, interpolate or otherwise substitute data values?
  • If you believe that specific factors will be more important than others in your analysis, you should indicate so now. You will use this to confirm your intuitions in your final write-up.

How to find data

There is a plethora of data resource these days. Here are a few notable (classic ML data) ones.
UCI ML data repository: Their data is mostly from researchers and is relatively clean. Also, the task types are mostly for classification or regression. Therefore many of them are suitable for the scope of this course's project.
Kaggle: Perhaps one of the most popular data science/ML data repositories today, they have many interesting ongoing or past competitions. But most of the recent datasets/tasks will be beyond the scope of this course. If you are still interested, consider choosing problems with tabulated data and classification or regression tasks.
Data.gov has many government sources of data. You can filter for a specific topic (e.g., Finance) and then restrict your attention to, e.g., CSV data, which should be easier to process.
